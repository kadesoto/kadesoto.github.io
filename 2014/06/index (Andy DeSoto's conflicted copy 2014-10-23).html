June 2014

Complete dataset for 18

Good news -- N = 599 subjects have been collected for my latest project (a cross-sectional study), which means we have a complete dataset. This is great news. New challenges arise, though. In particular, I am faced with issues regarding how to score the dataset. When I programmed the study, I implemented a pattern matching algorithm to score the data automatically. Unfortunately, it seems to be mishandling a few items selectively, leaving the options of (1) going with the data as scored, (2) hand-checking the troublesome items, or (3) re-matching all the data (which would be technically time-consuming -- I might need to do it in another programming language). Instead, I am leaning towards using an Excel matching that I've also programmed. Even though this algorithm is stricter, it carries the benefit that I know what it's doing -- it's completely transparent. This is really important.

There's one disadvantage with the Excel algorithm -- currently, the way it assigns credit is a little off, and it's something I'll need to change for the final version. This will require retooling the spreadsheet I currently use to score the data. Another disadvantage of Excel is that it's not the most sophisticated data management system I could be using. A few high-profile scientific errors involving Excel have popped up in the last few years, like this one. We clearly want to avoid those. Of course, I'll be publishing the raw data as soon as the paper's out so that others can confirm the results.

So the rest of the morning will be spent retooling this spreadsheet and going with the Excel scoring. I'm reasonably confident in this decision.
